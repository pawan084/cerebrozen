# iOS Parity Backport — Android Redesign → iOS

> Spec produced 2026-07-12 by reading `docs/REDESIGN.md` (findings F1–F11), `docs/TODO.md`
> ("Done — recent", redesign waves), the full iOS client (`apps/ios/CereBro/**`, ~10.6k LOC,
> all files read), and the shipped Android implementation (read-only). **Analysis only — no
> code changed.** Effort: S ≤ ½ day, M ≈ 1–2 days, L ≥ 3 days. Everything here is
> static-read only: this machine cannot run `xcodebuild`, so items marked ⚠ need a
> macOS build/verify pass.
>
> Headline: iOS is structurally *less* fragmented than pre-redesign Android (2 breathing
> implementations vs 4, one player screen vs 3 audio routes), so several Android waves
> shrink to copy/IA edits on iOS. The two genuinely large items are the **Dawn theme**
> (iOS is hard-pinned to dark) and the **Home/Toolkit IA restructure**.

## Deliberate divergences & decisions taken in this spec

- **Kill the four mini-games on iOS too** (item 2) — spec-consistent; revisit only with usage data.
- **Keep iOS real glass + aurora** (item 18) — Android's "fake glass" finding doesn't apply.
- **ToolAmbience stays mix-with-others** (item 5) — it's a bed under tools by design; only the
  soundscape engine needs single-instance discipline.
- **`SOSView` survives** as Talk's quick door (Android has no exact analogue) but reuses Toolkit tools.
- **Analytics before consent** (open Android decision in TODO) applies to iOS identically —
  `OnboardingFlow` tracks `onboarding_step` from Welcome. Same product/legal decision pending; not changed here.
